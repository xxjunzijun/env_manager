# UT 上库流程

## 规则

**任何代码上库前，必须先在本地运行单元测试（UT），确保全部通过后再提交。**

## 流程

```
1. 修改代码
       ↓
2. 运行 UT 测试 + 生成报告
       ↓
3. 所有测试通过？
   ├── 是 → 提交代码 → 推送到 GitHub
   └── 否 → 修复代码 → 回到第 2 步
              ↑____ 最多 5 次循环
```

## 具体步骤

### 第 1 步：运行测试

```bash
cd /home/node/.openclaw/workspace-coding/env_manager
python3 -m pytest tests/ -v
```

### 第 2 步：确认结果

- **78 passed** — 全部通过，继续
- **有 FAIL/ERROR** — 有测试失败，需要修复

### 第 3 步：生成报告（测试通过后）

将测试结果整理成报告，包含：
- 总测试数、通过数、失败数
- 各模块覆盖情况
- 失败测试的简要说明（如有）

### 第 4 步：提交与推送

```bash
git add -A
git commit -m "描述本次修改"
git push
```

## 修复循环（最多 5 次）

如果测试失败：

1. 查看失败原因
2. 修复对应代码
3. 重新运行测试
4. 再次确认结果

> 5 次内仍无法全部通过？暂停并报告问题给团队负责人。

## 示例报告格式

```
env_manager UT 测试报告
========================

总测试数: 78
通过: ✅ 78
失败: ❌ 0
耗时: 1.49s

[各模块测试结果...]
```

# UT 上库流程

## 规则

**任何代码上库前，必须先在本地运行单元测试（UT），确保全部通过后再提交。**

## 流程

```
1. 修改代码
       ↓
2. 运行 UT 测试 + 生成报告
       ↓
3. 所有测试通过？
   ├── 是 → 报告原因+措施 → 直接 commit + push
   └── 否 → 修复代码 → 回到第 2 步
              ↑____ 最多 5 次循环
```

## 具体步骤

### 第 1 步：运行测试

```bash
cd /home/node/.openclaw/workspace-coding/env_manager
python3 -m pytest tests/ -v
```

### 第 2 步：确认结果

- **87 passed** — 全部通过，继续
- **有 FAIL/ERROR** — 有测试失败，需要修复

### 第 3 步：汇报（测试通过后）

向用户说明：
- 错误原因
- 修改措施

通过后直接推送，**无需等用户确认**。

### 第 4 步：提交与推送

```bash
git add -A
git commit -m "描述本次修改"
git push
```

## 修复循环（最多 5 次）

如果测试失败：

1. 查看失败原因
2. 修复对应代码
3. 重新运行测试
4. 再次确认结果

> 5 次内仍无法全部通过？暂停并报告问题。

## 示例报告格式

```
UT 测试报告
========================
总测试数: 87
通过: ✅ 87
失败: ❌ 0
耗时: 1.49s

错误原因: ...
修改措施: ...
```

## 注意

- **UT 通过后直接推送**，不需要等用户确认
- 每次报错处理后都要汇报原因和措施
